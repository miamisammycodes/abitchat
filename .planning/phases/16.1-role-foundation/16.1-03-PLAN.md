## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anonymous web client → /admin/* | Crossed by every request to admin URLs; defended by 'auth' → 'require.super_admin' chain |
| anonymous webhook (Stripe, DK Bank) → handler | NOT user-authenticated by design; webhook signature verification is the trust gate, NOT RequireSuperAdmin |
| authenticated tenant user → /admin/* | Crossed when a tenant user tries to reach admin URLs; defended by RequireSuperAdmin's role check (returns 403, no info leak) |
| authenticated dual-role user → context switch | Crossed at /login/choose POST; defended by per-context role check in ChooseRoleController::store |

| T-16.1.03-04 | Elevation of Privilege | dual-role chooser POST | mitigate | ChooseRoleController::store rejects requests where user doesn't hold the requested role (abort 403). Single-role user with super_admin POSTing context=tenant → 403, and vice versa |
| T-16.1.03-05 | Information Disclosure | dropped `admin` guard residue | mitigate | `Auth::guard('admin')` calls in the application are tracked by SUMMARY for Plan 05 cleanup (AdminActivityLog::log + any controller that still references the guard); without cleanup, runtime errors surface but never silently grant access |
