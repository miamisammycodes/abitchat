<objective>
Rewrite the database schema and User model to support multi-role per user via a `user_roles` pivot table. Drop the `admin_users` table + the `users.role` enum column. Re-point the two foreign keys (`transactions.approved_by`, `admin_activity_logs.admin_user_id`) at `users(id)`.

Purpose: This is THE migration plan — the entire phase's downstream work (Gates, policies, Inertia share, Vue v-if sweep, seeder) depends on the new schema. Per RESEARCH Pitfall 1, the 8-step migration sequence is non-negotiable: FK drops MUST precede table drop; FK re-adds happen in fresh `Schema::table()` calls; column drops/alters are separate migrations. No prod data exists (per D-05) so `migrate:fresh --seed` is the validation target.

Output: 8 migrations + 1 new model (UserRole) + 3 modified models (User, Transaction, AdminUser-deletion).
</objective>
