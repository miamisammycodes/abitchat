## UpdateWebsiteIndexingRequest Trigger Resolved

This CLOSES the issue that opened Phase 16.1:

**Before (broken):**
```php
public function authorize(): bool {
    return $this->user()?->isOwner() ?? false;  // isOwner() deleted in Plan 02
}
```

**After (fixed):**
```php
public function authorize(): bool {
    return Gate::allows(Ability::ManageTenantSettings->value);  // Owner-tier gate
}
```

Matrix behavior:
- Owner → `Gate::allows('manage-tenant-settings')` → true → 302 success
- Manager → false → 403
- Agent → false → 403
- Unauthenticated → auth middleware redirects to /login before FormRequest runs
