# Phase 16.1 Plan 06a: Inertia Share Rewrite + Vue Role Chrome Summary

**HandleInertiaRequests::share() rewritten to emit a unified auth.user payload with 13-key ability map, role flags, and URL-context-aware primary_role — plus three new Vue role-chrome components (RoleBadge, ContextSwitchChip, ChooseRole) and layout updates.**

- **Tasks:** 2 (both TDD: RED→GREEN commits)

- **Files created:** 3 Vue components
- **Files deleted:** 1 (Admin/Auth/Login.vue)

## Accomplishments

### Task 1: HandleInertiaRequests::share() Rewrite

- Rewrote `share()` to emit unified `auth.user` shape: `id`, `name`, `email`, `tenant_id`, `can`, `roles`, `has_super_admin_role`, `has_tenant_role`, `primary_role`
- Added `buildAbilityMap(User $user): array<string, bool>` — iterates all 13 `Ability::cases()`, converts hyphenated slugs to snake_case, checks `Gate::forUser($user)->check()` per request (no cache)
- Added `buildAuthUser(User $user, Request $request)` — unified user payload builder
- Added `resolvePrimaryRole(User, Request, bool)` — URL context logic: `admin/*` + `has_super_admin_role` → SuperAdmin; else highest tenant role by `rank()`; null fallback
- Added `loadMissing('roles')` before emitting — single user_roles query guaranteed (N+1 protection)
- Removed `auth.admin` key entirely (not nulled — key does not appear at all)

### Task 2: Vue Role Chrome Components

- **RoleBadge.vue**: coloured pill badge (violet=super_admin, blue=owner, sky=manager, slate=agent); consumes `primary_role: {value, label}` prop
- **ContextSwitchChip.vue**: switch link visible only when user has both `has_super_admin_role` and `has_tenant_role`; links admin→/dashboard, client→/admin/dashboard; uses Inertia `<Link>`
- **ChooseRole.vue**: dual-role context chooser; renders `availableContexts` from controller; posts to `/login/choose`; card-based UI with Shield/LayoutDashboard icons
- **AdminLayout.vue**: replaced `const admin = computed(() => page.props.auth?.admin || {})` with `const user = computed(() => page.props.auth?.user ?? null)` and `primaryRole` computed; swapped `admin.name` + `<Badge>{{ admin.role }}</Badge>` with `{{ user?.name }}` + `<RoleBadge :role="primaryRole" />`; added `<ContextSwitchChip :user="user" context="admin" />`; removed unused `Badge` import
- **ClientLayout.vue**: added `primaryRole` computed; inserted `<ContextSwitchChip :user="user" context="client" />` as first child of right header cluster; inserted `<RoleBadge :role="primaryRole" />` before user name span
- **Deleted**: `resources/js/Pages/Admin/Auth/Login.vue` (legacy admin guard — removed in Plan 03)
