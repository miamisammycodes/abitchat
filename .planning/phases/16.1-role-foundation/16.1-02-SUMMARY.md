# Phase 16.1 Plan 02: Database + Model Foundation Summary

**One-liner:** 8-step ordered migration sequence drops admin_users, pivots to user_roles, rewires User model with hasRole(Role)/hasRoleAtLeast(Role,Tenant), and wires SeedsRoleMatrix fluent helpers.

## What Was Built

### Migration Sequence (8 migrations, strict order)

The migration sequence was required to avoid FK constraint violations:

1. **000001** — Drop `transactions.approved_by → admin_users` FK
2. **000002** — Drop `admin_activity_logs.admin_user_id → admin_users` FK
3. **000003** — Drop `admin_users` table (with reversible `down()`)
4. **000004** — Re-add `transactions.approved_by → users` FK (nullOnDelete)
5. **000005** — Re-add `admin_activity_logs.admin_user_id → users` FK (cascadeOnDelete)
6. **000006** — Drop `users.role` column
7. **000007** — Make `users.tenant_id` nullable (for SuperAdmin platform users)
8. **000008** — Create `user_roles` pivot table

`user_roles` schema:
```
id, user_id (FK→users, cascade), role (enum: super_admin|owner|manager|agent),
tenant_id (nullable FK→tenants, cascade), timestamps
indexes: (user_id, tenant_id), (tenant_id, role)
unique: (user_id, role, tenant_id) — named user_roles_user_role_tenant_unique
```

### UserRole Model Interface

```php
namespace App\Models;

/**
 * @property Role $role
 */
class UserRole extends Model
{
    use BelongsToTenant;
    // fillable: user_id, role, tenant_id
    // casts: role => Role::class (backed enum)
    public function user(): BelongsTo // BelongsTo<User, $this>
}
```

### User Model Interface (Plan 03+ depends on these signatures)

```php
namespace App\Models;

/**
 * @property-read Collection<int, UserRole> $roles
 */
class User extends Authenticatable
{
    // fillable: name, email, password, tenant_id (role REMOVED)
    public function roles(): HasMany // HasMany<UserRole, $this>

    // Exact match for platform-level roles; tenant-scoped for others.
    // SuperAdmin check ignores $tenant (platform-level).
    public function hasRole(Role $role, ?Tenant $tenant = null): bool

    // Rank comparison within tenant. Excludes platform-level roles entirely.
    // Owner(3) >= Manager(2) >= Agent(1). SuperAdmin(0) always returns false.
    public function hasRoleAtLeast(Role $min, Tenant $tenant): bool

    // DELETED: isOwner(), isManager(), isAgent(), hasRole(string)
}
```

### SeedsRoleMatrix Trait Interface

```php
trait SeedsRoleMatrix
{
    protected ?User $superAdminUser;
    protected ?User $ownerUser;
    protected ?User $managerUser;
    protected ?User $agentUser;

    public function actingAsSuperAdmin(): static         // creates User + SuperAdmin pivot, actingAs()
    public function actingAsOwner(Tenant $tenant): static    // creates User + Owner pivot, actingAs()
    public function actingAsManager(Tenant $tenant): static  // creates User + Manager pivot, actingAs()
    public function actingAsAgent(Tenant $tenant): static    // creates User + Agent pivot, actingAs()
}
```

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] APP_BASE_PATH required in phpunit.xml for worktree test isolation**
- **Found during:** Task 1 (migration tests not picking up new migrations)
- **Issue:** `Application::inferBasePath()` resolves path from Composer's registered `ClassLoader`. Since `vendor/` is a symlink to `/Users/sam/Dev/laravel/chatbot/vendor`, PHP resolves the autoloader path to the main repo, causing all tests to load `bootstrap/app.php` from the main repo (which uses the main repo's migrations directory, not the worktree's).
- **Fix:** Added `<env name="APP_BASE_PATH" value="/path/to/worktree"/>` to `phpunit.xml`. This is picked up by `Application::__construct()` before `inferBasePath()` is called.
- **Files modified:** `phpunit.xml`
- **Commit:** `40ee80a` (included in GREEN Task 1)

**2. [Rule 3 - Blocking] Tenant::factory()->create() required instead of raw DB insert in pivot shape test**
- **Found during:** Task 1 (UserRolesPivotShapeTest)
- **Issue:** Raw `DB::table('tenants')->insertGetId([...])` failed with `NOT NULL constraint failed: tenants.slug`. The `tenants` table has a `slug` column that is NOT NULL, which a raw insert without factory defaults cannot satisfy.
- **Fix:** Changed to `Tenant::factory()->create()` which handles `slug` and all other required columns via factory defaults.
- **Files modified:** `tests/Feature/Migrations/UserRolesPivotShapeTest.php`
- **Commit:** `40ee80a`

**3. [Rule 1 - PHPStan] @phpstan-ignore-next-line annotations removed from RolePermissions.php**
- **Found during:** Task 3 (User model rewrite)
- **Issue:** Plan 01 had added two `@phpstan-ignore-next-line` annotations as forward-declarations for `User::hasRole(Role)` and `User::hasRoleAtLeast(Role, Tenant)`. After Plan 02 adds these methods, the annotations must be removed to maintain baseline=0.
- **Fix:** Removed both annotations.
- **Files modified:** `app/Auth/RolePermissions.php`
- **Commit:** `2c3d7e5`

**4. [Out-of-scope] UpdateWebsiteIndexingRequest.php isOwner() error NOT added to PHPStan baseline**
- **Noted during:** Task 3 verification
- **Decision:** `app/Http/Requests/Client/UpdateWebsiteIndexingRequest.php:14` calls `$this->user()?->isOwner()` which no longer exists. This file is NOT touched by Plan 02. PHPStan baseline=0 success criterion is scoped to touched files per project conventions. Plan 04 will fix this call site with a Gate check.
- **Action:** Documented in broken call sites section above. No baseline.neon entry added.
