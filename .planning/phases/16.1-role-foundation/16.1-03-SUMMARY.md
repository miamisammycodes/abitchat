# Phase 16.1 Plan 03: Single-guard auth collapse + login routing Summary

Single `web` guard replaces dual `web`+`admin` guard; `RequireSuperAdmin` middleware gates all `/admin/*` routes; `Auth\LoginController::store` branches post-login into four role-aware cases; `ChooseRoleController` handles the dual-role chooser at `/login/choose`.

## What Was Built

### Task 1 — RequireSuperAdmin middleware + config/auth.php collapse

**config/auth.php (single-guard confirmed):**
- Removed `admin` guard (was `driver: session, provider: admin_users`)
- Removed `admin_users` provider and `admin_users` password broker
- Only `web` guard + `users` provider + `users` password broker remain
- `auth.defaults.guard = 'web'` is the sole active guard

**RequireSuperAdmin middleware (new):**
- Path: `app/Http/Middleware/RequireSuperAdmin.php`
- Alias registered in `bootstrap/app.php` as `require.super_admin`
- Logic: `abort(403)` if user is not an instance of `App\Models\User` or does not have `Role::SuperAdmin`
- Intentionally does NOT redirect — `auth` middleware upstream handles the 302 for unauthenticated requests

**AdminAuthenticate.php (deleted):** Legacy dual-guard middleware removed entirely.

### Task 2 — routes/web.php rewire + ChooseRoleController + webhook safety

**routes/web.php changes:**
- Admin group: `Route::prefix('admin')->name('admin.')->middleware(['auth', 'require.super_admin'])->group(...)`
- `admin.logout` now points to `Auth\LoginController::destroy` (single controller)
- `/login/choose` GET + POST routes added inside `auth` middleware group (names: `login.choose`, `login.choose.store`)
- Admin-specific login routes (`admin/login`) deleted — single `/login` entry point
- Comment added documenting Pitfall 7: Cashier auto-registers `/stripe/webhook` outside this group

**Stripe/Cashier webhook route confirmed OUTSIDE `require.super_admin`:**
- URI: `POST stripe/webhook` → `cashier.webhook`
- Middleware: none (Cashier handles its own HMAC signature verification internally)
- `/stripe/payment/{id}` also has no custom auth middleware
- Neither Stripe route appears in the `admin.*` namespace

**ChooseRoleController (new):**
- `create(Request $request)`: detects dual-role status using `$user->hasRole(Role::SuperAdmin)` + `roles->contains(!isPlatformLevel())`; redirects single-role users immediately (no chooser shown); renders `Inertia::render('Auth/ChooseRole', ['availableContexts' => [...]])`
- `store(Request $request)`: validates `['context' => ['required','string','in:admin,tenant']]`; aborts 403 if user lacks the requested role; redirects to `admin.dashboard` or `dashboard` accordingly

### Task 3 — LoginController role-aware redirect + all test suites

**Auth\LoginController::store redirect logic (4 cases):**
```
$isSuperAdmin && $hasTenantRole  → route('login.choose')
$isSuperAdmin                    → route('admin.dashboard')
$hasTenantRole                   → route('dashboard')
default (no roles)               → '/' with error flash
```
- `$user->loadMissing('roles')` guards against N+1 (Pitfall 5)
- `$hasTenantRole` uses `!$ur->role->isPlatformLevel()` to detect any tenant-scoped role
- `$request->session()->regenerate()` preserved before role check (ASVS anti-fixation)
