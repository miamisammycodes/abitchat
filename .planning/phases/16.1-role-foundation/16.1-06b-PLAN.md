Thread `v-if="$page.props.auth.user.can.X"` across every Client Vue page that exposes role-gated mutation controls, per the canonical per-page mapping table. Ship the R4 empty state for `Analytics/Index.vue` (Agent role sees the locked "No actions available" panel per UI-SPEC R5). The key-alignment test exists because Vue template typos otherwise fail silently (the literal `can.manage_billings` is `undefined`, JavaScript coerces to falsy, and the control simply never appears for anyone — a coverage gap that the matrix tests don't catch).

Output: 16 modified Client Vue pages + 1 new test file (the key-alignment guard). Admin pages are NOT modified — they're already gated by RequireSuperAdmin middleware.
