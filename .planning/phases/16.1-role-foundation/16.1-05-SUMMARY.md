## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Worktree class loading — tests ran against main repo code**

- **Found during:** Task 1 (authorize calls not blocking managers — controllers returned 302 instead of 403)
- **Issue:** Composer's classmap in vendor/composer/autoload_classmap.php maps all `App\*` classes to main repo's app/ directory. The worktree shares the parent repo's vendor/ directory, so without intervention, all PHP class loads resolve to main repo code (pre-authorize-call controllers).
- **Fix:** Created `tests/bootstrap.php` using PHP Reflection to strip `App\*` and `Database\*` from the classmap, then set PSR-4 `'App\\'` to the worktree's app/ directory. Updated `phpunit.xml` to use `bootstrap="tests/bootstrap.php"`.
- **Files:** tests/bootstrap.php (new), phpunit.xml (modified)
- **Commit:** 5d49d77 (included in Task 1 GREEN)

**2. [Rule 2 - Critical] Queue::fake() in AbilityCoverageTest positive-access knowledge base test**

- **Found during:** Task 1 test implementation
- **Issue:** `test_manager_can_create_knowledge_item` causes `ProcessKnowledgeItem` job to dispatch synchronously, failing with embedding provider error (ollama not running in test environment).
- **Fix:** Added `Queue::fake()` before the POST request in that specific test.
- **Files:** tests/Feature/Client/AbilityCoverageTest.php
- **Commit:** 5d49d77

**3. [Info] AdminActivityLog::log() already correct — Task 3 is test-only**

- AdminActivityLog::log() was already rewritten in a prior commit to use `Auth::user()` instead of `Auth::guard('admin')->user()`. The plan's Task 3 implementation step was already done. Task 3 delivered 4 tests that lock the invariant.
