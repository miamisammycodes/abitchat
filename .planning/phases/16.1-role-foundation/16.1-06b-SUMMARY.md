## Deviations from Plan

### Plan-Reality Mismatches (plan described future state, not current UI)

The plan's per-page mapping table listed controls that do not yet exist in the codebase:

| Page | Plan Said | Reality |
|------|-----------|---------|
| Dashboard.vue | "Upgrade plan CTAs, manage-team prompts" | No such CTAs exist; only GET nav links |
| Conversations/Index.vue | "Delete buttons, bulk actions" | No delete or bulk action controls |
| Billing/Index.vue | "Upgrade, Cancel, Reactivate CTAs" | Only GET nav links (View Plans, Renew Now) |
| Billing/Transactions.vue | "Download invoice, Refund request links" | Only GET download links (R7: not gated) |

**Resolution:** Skipped v-if additions for non-existent controls. Zero changes to these 4 files. Plan 07 browser smoke (D-20) will surface if any of these controls exist post-merge.

### Conversations/Show.vue: v-else-if for Unarchive (Rule 2)

Plan said to gate Archive and Unarchive with `v-if`. The original used `v-if/v-else` based on conversation status. Changed to `v-if/v-else-if` (both with the ability gate) to prevent the Archive button from showing for an unarchived conversation when the user lacks `manage_conversations`. This is the correct behavior: neither button shows if the user can't perform the action.

### Billing/Plans.vue: Subscribe button v-else-if tightened (Rule 1 bug fix)

Original Subscribe button used bare `v-else`. After gating Contact Us and Free Trial with `&& can.manage_billing`, the v-else chain would fall through to Subscribe for enterprise plans when the user lacks billing permission (because Contact Us condition was false). Fixed by changing v-else to `v-else-if="! plan.is_contact_sales && plan.price > 0"` — prevents Subscribe from showing on enterprise plan cards.

### Test uses class-based PHPUnit (not Pest `it()`)

Plan said "Pest test style consistent with tests/Feature/Inertia/". The only existing test (AuthShareTest.php) is class-based PHPUnit. No Pest tests exist in the project. PHPStan would flag `it()` and `expect()` as undefined functions without the Pest PHPStan extension. Decision: class-based PHPUnit for project consistency and clean static analysis.

### npm build runs from main repo (not worktree)

Worktree has no `vendor/` directory (pre-existing limitation — worktrees share the main repo's vendor). `npm run build` was validated from `/Users/sam/Dev/laravel/chatbot` (green) after Task 1 merges via git. The worktree build failure is pre-existing and not caused by this plan's changes.
